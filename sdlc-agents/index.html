Plan · Estimate · Spec

Architecture & Planning Agent

Answers "what will this take?" with evidence pulled from real code, Arch Web, and Confluence — not a gut feel.

What it does & impact

  • Grounds every plan. Scans GitLab (no clone), Arch Web, and Confluence for real evidence.
  • Dual estimates. Effort with an AI agent vs without, per work item and total.
  • Confidence score. Every plan carries a High / Medium / Low %, tied to the evidence scanned.
  • Finds shared components. Across EValue, MedHub, ATI before anything is rebuilt.
  • Build-ready specs. Emits an OpenSpec a coding agent implements directly.
  • Impact. Estimation becomes evidence, not guesswork — the AI productivity delta is a number leaders can see.
~40%lower effort on boilerplateIllustrative
Model: Claude Sonnet 4 · Scans GitLab (no clone) · Arch Web · Confluence — every claim source-attributed.

Build · Test · Review

MedHub Coding Agent

The MedHub agent family — five custom Copilot agents sharing one skill system — carries a ticket from intake to closure, pre-loaded with MedHub architecture, standards, and workflows.

What they did & impact

  • Five agents, one skill system. ambs-debug, medm-analysis, medm-implement, review-mr, datadogagent.
  • AMBS bug-fixing. Open production bugs cut from ~1,000 to ~200 over recent months.
  • QE / accessibility. ~50,000 accessibility issues fixed in ~1.5 months (WCAG 2.1 AA, axe-core).
  • Full ticket lifecycle. Investigation → implementation → tests → review → Jira closure.
  • Pre-loaded MedHub context. Architecture, standards, and workflows built in — nothing re-explained.
  • Guardrails at every write. Preview before apply, no auto-merge, SELECT-only SQL.

Observe · Attribute · Remediate

Production Triage Agent

Collapses a five-tool HTTP-500 investigation into a single chat — no tab switching, no copy-pasting between Datadog, the IDE, the terminal, and GitLab.

What it does & impact

  • One conversation. Replaces a five-tool HTTP-500 investigation.
  • Observe. Correlates Datadog APM traces and logs by trace ID.
  • Regression flagged. When an error is first seen ≤30 min after a deploy.
  • Attribute. git blame names the introducing commit, author, and MR automatically.
  • Remediate. Preview-first fix; commit and MR only on your approval.
  • Documents & proven. Markdown report + duplicate-checked Jira ticket; two demo sessions recorded with the MedHub team.
3.2×faster triage — alarm to fixIllustrative
Skills: datadog-analysis · git-analysis · bug-fix · Two demo sessions recorded with the MedHub team (Feb & Apr 2026).
